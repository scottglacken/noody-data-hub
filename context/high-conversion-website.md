## 1. Core CRO Framework

### The Revenue Per Visit (RPV) Diagnostic
- **Key metric**: Revenue Per Visit = Conversion Rate x AOV
- **Benchmark**: $3.00 RPV across 600+ brands (2023 data)
- If RPV < $3, conversion is likely your bottleneck -- prioritise website work
- If RPV > $3, bottleneck is probably traffic/marketing, not website
- Don't pursue full website rebuilds -- they're high effort, low impact
- The right question: "How do we get more people to take the next step?" (not "how do I make it better")

### The HCW System (9 Modules)
Three pillars:
1. **Design** -- Make site compelling (home, collection, PDP, cart key elements)
2. **Streamline** -- Remove friction (page speed, policies/payments, screen recordings to find rage clicks)
3. **Sell** -- Turn website into a salesperson (landing pages, split testing, better imagery/copy)

### The GAP Analysis Method
- Audit each page element: Do you have it? (Y/N) -- Score out of 10 -- Do you need it?
- 19 elements for homepage, separate audits for PDP and collections
- Creates a one-page action plan of what to fix
- Not every business needs every element -- assess based on product type

### Development Planning
- **When to update**: RPV below $3.05, low conversion rate, product-market fit confirmed
- **Three paths**: Keep current theme (cheapest), new Shopify 2.0 theme ($500-2k), blank canvas ($15-50k+ for $10M+ businesses)
- **Validate with apps first**, then customise if they work -- avoids ongoing fees and conflicts
- **Tackle one page at a time** -- don't rebuild everything at once
- Plan for a theme refresh every ~3 years
- **Customisation costs**: $2-5k per major page (homepage/PDP/mini cart), $2-3k for collections, $1-2k design per template

## 2. Homepage Optimisation

### Above the Fold (5 Key Elements)
1. **Promo bar** -- Clear, testable offers (free shipping thresholds, current promos)
2. **Navigation** -- Visual, clear categories; prioritise site search for complex catalogues
3. **Hero banner** -- Flexible (image, video, personalised); test different formats
4. **Secondary navigation** -- Curated collections or category tiles
5. **Email capture** -- Right timing and compelling offer (not just "sign up for 10% off")

### Below the Fold (Optional Blocks)
- Primary text block (mission statement / #1 USP)
- Featured products / best sellers
- Credibility section (as seen in, press logos, certifications)
- "Reasons why" panel with icons
- Video / UGC / community validation
- Curated reviews (not just auto-pulled)
- Blog / founder story
- SEO content block
- Smart footer with key links
- Keep page long but clean above-the-fold; test everything

### Homepage Principles
- Homepage = landing page -- treat it as one
- Use CTAs with tested, compelling wording (not generic "Shop Now")
- Replace "quiz" with benefit-driven copy like "Find the right light"
- Add video/UGC/community validation for trust and engagement

## 3. Product Detail Page (PDP) Optimisation

### Above the Fold (Buy Box)
- **Most important page on the site** -- above fold should act like a landing page
- Show immediately: product name, USP/problem solved, strong review snippet
- Use space wisely on mobile -- every pixel counts
- Mix polished product shots + UGC in image gallery
- Simplify add-to-cart and subscription blocks
- Highlight promos above the fold
- Image format: 1:1 ratio safest, use WebP format
- Test colour swatches + sticky CTA options
- Show per-unit pricing and savings tiers for quantity selectors

### Below the Fold
- **Don't stop at reviews + related items** -- extend PDP like a landing page
- Add: icon panels, "reasons why", product demo video, FAQs, founder video
- **FAQs must be sales-focused**: product-specific, elevator-pitch style (not generic shipping Q&A)
- Use curated/hand-picked reviews (not just chronological)
- Community validation section
- Bundled/related products below
- Shopify templates make adding these blocks easier

### PDP Testing Priorities
- Product image variations (lifestyle vs model vs flat lay)
- Price elasticity testing (+/-10-15%)
- Layout changes (buy box arrangement)
- Bundle builders for higher AOV
- Sticky add-to-cart button on mobile

## 4. Collection Page Optimisation

### When Collections Matter
- Critical for large SKU / fashion brands
- Less important for small product ranges (<20 SKUs)

### Collection Page Elements
- Skip big hero banners -- use the space for offers/promos and push products higher
- **Must-haves**: clear titles, "in stock only" filter, strong filtering/sorting
- Enhance with: subcategories, curated images, promo boxes, quick add-to-cart
- For scale: use listicle-style collections as landing pages
- Collection sorting: <20 SKUs = optional; >20 SKUs = keep sorting options
- Show 1-2 products per row on mobile with extra info (ratings, key benefit)

### Promotional Collection Pages
- Variant of collection template designed as a landing page
- Increases both CVR and AOV
- Include offer banner, navigation to sale collections, best sellers with discounts
- Show actual percentage off rather than just "sale"

## 5. Cart & Checkout Optimisation

### Mini Cart / Cart Drawer
- Show gift-with-purchase thresholds clearly with dollar amounts
- Integrate upsell suggestions within the cart
- Test cart drawer vs full cart page
- Show shipping threshold progress bar ("Add $X for free shipping")

### Checkout
- Don't customise checkout with code -- use apps instead (cheaper, more reliable)
- Checkout experiments (Shopify Plus): shipping rates, components, trust elements
- Optimise checkout abandonment with retargeting and email flows
- Reviews on checkout page -- worth testing
- Never use "final sale, no returns" -- violates ACCC/FTC requirements
- Update T&Cs for seasonal events: shipping cutoffs, extended gift return windows

## 6. Landing Page Framework

### Five Questions Every Landing Page Must Answer
1. Why are you selling this? (the problem/angle)
2. What problem does it solve?
3. How does it compare to alternatives?
4. When will it arrive? (shipping/delivery)
5. What do others think? (social proof/reviews)

### Landing Page Principles
- Each ad = one problem/angle; Meta sells the click, landing page completes the journey
- Always show an offer
- Don't lead with product names -- focus on the solution
- Aim for weekly new landing pages to build a library
- Ad messaging and landing page must match in imagery, messaging, and tone
- Advertised products must be visible within first couple of scrolls

### Three Landing Page Formats to Test
1. **5 Reasons** -- Five compelling reasons to buy (most recommended)
2. **Listicle** -- Editorial-style list format
3. **Editorial** -- Story/content-driven approach

### The "Ugly is Pretty" Principle
- Organic and genuine designs outperform polished designs
- Low-fi, founder-led content often converts better
- Don't over-design -- test simple layouts first

### Landing Page Structure
- **Above fold**: Hero with offer, strong headline addressing the problem, primary CTA
- **Below fold**: Social proof, reasons why, product details, comparison, FAQs, secondary CTA
- Include offer banner at top, navigation to relevant collections, best sellers with discounts

### Implementation Options
1. Build custom sections with a developer
2. Page builders: Replo, GemPages, Zipify, PageFly
3. Shopify native pages with Section Store app for drag-and-drop sections

## 7. Page Speed & Core Web Vitals

### The Reality of Page Speed
- Core Web Vitals (LCP, CLS, INP) are ONE of many Google ranking signals -- not the only one
- Even Shopify's Dawn demo theme doesn't get a perfect score
- Modern mobile speeds (90-98 Mbps avg) are far more forgiving than 2016 (17 Mbps)
- Less than 3% of AU market on 3G connections now
- **Don't chase perfect scores** -- if it loads well on your devices, it's likely fine
- Page speed score does NOT have a direct correlation with conversion performance

### What Actually Slows Your Site
- Every app adds code/complexity
- Every new section adds load
- Legacy code from multiple developers over time
- Uncompressed images and videos
- Too many third-party scripts

### Speed Optimisation Checklist
- Remove unused/disabled apps (they often leave code behind)
- Compress images (use WebP format, apps like Crush Pics)
- Use preview loops for videos rather than auto-playing full videos
- Choose web-safe fonts
- Lazy load below-fold content
- Audit apps regularly -- validate they're adding value
- Use Booster app for technical speed improvements
- **Focus on practical fixes, not vanity scores**

## 8. A/B Testing Framework

### Fundamentals
- AB testing eliminates seasonality bias -- control and variation run simultaneously with split traffic
- Require **95% statistical significance** to determine if results are real
- Run tests for **minimum 1 week** with **100-200 orders per variation**
- Control risk by adjusting traffic allocation or stopping underperforming tests immediately

### Planning & Prioritisation (ICE Framework)
- Score experiments using: **Impact, Confidence (effort), Ease, and Traffic strength**
- Back experiments with data (analytics, heatmaps, surveys, competitor research) for 90% win rate
- **Measure the right metrics per page**:
  - Homepage: engagement / bounce rate
  - PDP: add-to-cart rate
  - Checkout: purchase completion rate

### Testing Strategy
- Test **small isolated changes** rather than wholesale redesigns -- faster, clearer results
- Compound small wins: each 2-5% improvement builds on the previous baseline
- One change per test -- isolate variables
- DIY: 4-50 hours per experiment; Agency: 2k-15k/month
- Implement changes systematically with minimum one week between tests

### High-Impact Tests to Run
- Price elasticity (+/-10-15%)
- Product image variations (lifestyle vs model vs flat lay)
- Page-level changes (layout, structure)
- Shipping threshold amounts
- Checkout component variations (Shopify Plus)
- Hero banner copy and imagery
- CTA button text and placement
- Navigation structure

### AB Testing Tools (Shopify)
- **Elevate** -- Simple AB testing (page, price, theme, visual editor, split URL experiments)
- **Intelligems** -- Advanced AB testing
- **Shoplift** -- A/B testing app
- Shopify native AB testing coming (Winter 2025 edition)

## 9. Copywriting That Sells

### Common Mistakes
- Generic, feature-focused copy that doesn't speak to the customer's problem
- Using overused phrases ("premium quality", "shop now", "best in class")
- Not doing customer research before writing

### Great Copy Checklist
- Speaks to one specific problem or pain point
- Uses customer language (from reviews, surveys, support tickets)
- Focuses on benefits and outcomes, not features
- Includes social proof and endorsements where possible
- Has clear, compelling CTAs

### User Research Framework
- **Simple methods**: Read your own reviews, competitor reviews, Amazon reviews, support tickets
- **Advanced methods**: Customer surveys, Typeform, post-purchase surveys (KnoCommerce, Grapevine)
- Create personas based on research
- Test content variations (A/B test headlines, CTAs, product descriptions)
- Use AI to generate fresh copy focused on customer problems
- Power keywords that drive action

## 10. Trust Signals & Social Proof

### Review Optimisation
- Compelling reviews impact CVR more than almost any other PDP section
- Problem: reviews are typically buried at the bottom of the page
- **Five actionable adjustments**:
  1. Surface a strong review snippet above the fold in the buy box
  2. Curate best reviews (hand-pick, don't just show chronological)
  3. Use photo/video reviews prominently
  4. Add review quotes as standalone trust sections between content blocks
  5. Test review placement and format
- Review apps: Judge.me, LOOX

### Other Trust Signals
- "As seen in" / press logos
- Certifications and awards
- Founder story / video
- Community validation (UGC, customer photos)
- Shoppable video (Tolstoy, Videowise, Firework)
- "Wall of Love" section -- curated customer testimonials
- Shipping/returns policy visibility
- Payment method icons

## 11. Mobile UX Best Practices

### Mobile-First Design
- Most ecommerce traffic is mobile -- optimise for mobile first
- Every pixel of mobile screen space matters above the fold
- Use sticky CTAs (add to cart button that follows scroll)
- 1:1 image ratio is safest for mobile product galleries
- Test 1 vs 2 products per row on collection pages
- Declutter bundle pages on mobile
- Present key product information concisely

### Mobile Navigation
- Add prominent button in mobile header for sales/promos
- Predictive search results in mobile search
- Clean, visual hamburger menu with curated categories
- Mobile-optimised pop-ups (timing and size matter)

## 12. AOV Growth Tactics

### Bundling
- Bundle builders significantly increase AOV
- Apps: UpNova, Biscuit Bundles, EB Easy Bundle Builder, Pumper Bundles
- Modified collection pages can work as simple bundle pages (no app needed)
- Show per-item pricing and total savings clearly

### Upsells
- Mini cart upsells (Monster Cart, UpCart, Unicorn)
- Post-purchase upsells (One Click Upsell, AfterSell)
- Gift with purchase (GWP) at clear thresholds
- Shipping protection toggle (Navidium) -- test auto-on

### Subscriptions
- Apps: Loop, Recharge, Appstle, Seal, Stay AI, Skio
- Theme must support subscription blocks properly
- Simplify the subscription UI in the buy box

### Promotions Without Margin Loss
- GWP often outperforms straight discounts
- "Rush order" upsells for made-to-order products
- Size mapping and convenience upsells
- Redline pricing vs auto-applied discounts
- Avoid stacking discounts (disable sign-up pop-ups during major sales)

## 13. Conversion Funnel Diagnosis

### Shopify Funnel Report
- Track user behaviour: homepage > collections > PDP > cart > checkout
- Understand bounce rates, add-to-cart rates, conversion by page template and traffic source
- Use YOUR site's data as the benchmark, not generic best practices

### Diagnosis Framework
- **High bounce rate** (top of funnel): Fix hero images, value proposition, page load speed
- **Low add-to-cart** (PDP problem): Fix product clarity, benefits, imagery, trust signals, urgency
- **Cart abandonment** (checkout friction): Fix shipping costs, returns policy, payment options, trust at checkout
- Track marketing attribution to understand which channels drive quality traffic

### Data & Analytics Tools
- Shopify Reports -- create funnel performance reports
- Google Analytics 4 (GA4) -- deeper behaviour analysis
- Microsoft Clarity -- free heatmaps and session recordings (install manually, not via app -- app has bugs)
- Heatmaps: identify where people rage-click, get frustrated, or bounce
- Session recordings: watch real user journeys to find friction points

## 14. Search & Discovery

### Site Search
- Shopify Search & Discovery app for filters, recommendations, search rules
- Predictive search results improve mobile UX
- Curated search rules for key terms
- Track what people search for -- reveals product demand and navigation gaps
- Critical for complex catalogues with many SKUs

### Navigation Best Practices
- Visual navigation with curated category images
- Benefit-driven labels ("Find the right [product]" not "Take our quiz")
- Add sale/promo link in standout colour during events
- Subcategory navigation within collection pages

## 15. Seasonal / Black Friday Checklist

### Pre-BFCM Preparation
- Update pop-ups to announce BF sales and build email/SMS list (email list converts at 200%+)
- Ensure ad and landing page messaging match (imagery, tone, products)
- Add BF collection link to main nav in standout colour
- Update copy -- avoid overused phrases, use AI for fresh angles
- Update T&Cs: shipping cutoffs, extended gift return windows
- Test discount stacking (disable sign-up pop-ups to avoid compounding discounts)
- Create backups (Rewind, Theme Access for developers)

### During BFCM
- Monitor: site speed, conversion rates, inventory, cart abandonment
- Use Shopify Sidekick AI for quick analytics summaries
- Protect against bots/fraud (Signifyd, Cloudflare for IP blocking)

### Landing Page for Sales
- Include: offer banner at top, navigation to sale collections, best sellers with discounts
- Show actual percentage off (not just "sale")
- Daily deal landing page format
- Homepage structure adapted for sale period

## 18. Web Accessibility (ADA/WCAG)

### Core Criteria (WCAG)
- **Perceivable**: Alt text on images, captions on videos, sufficient colour contrast
- **Operable**: Keyboard navigable, no time-dependent actions
- **Understandable**: Clear language, consistent navigation, error prevention
- **Robust**: Works across browsers and assistive technologies

### Practical Guidance
- Compliance tools don't guarantee legal protection but show good faith effort
- If you receive a demand letter: don't panic, don't respond immediately, consult legal counsel
- Most ADA threats don't result in actual lawsuits
- Document compliance efforts as evidence of good faith
